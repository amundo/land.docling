About this course

The web platform is a complex system which has evolved over decades. Learning its ins and outs can seem overwhelming.

In this course you will be guided through just enough of the details of the web platform to understand what it is, how it works, and what it can do. While it would be unrealistic to expect that a single course could turn you into a professional web developer, you can learn to participate in the development of software for language documentation and description. Developing software, after all, involves much more than just writing software itself: at least as much work goes into specifying what software should do and how it should work as it does into the actual code.

Learning to specify these whats and hows in detail is a super power. By practicing this skill, you will become equipped participate in the development of software that meets your needs as a linguist.

The course is arranged into Lessons and Skills. Lessons are organized around building documents and applications that support the creation of digital versions of the “Boasian trilogy” — lexical, textual, and grammatical materials. Skills are bite-sized tutorials for the various technical skills that are applied in the creation of those documents and applications. Skills are largely self-guided, but you are welcome to discuss your experiences or any difficulties in the Docling forum.

Designing digital documentation with the Web Platform

Creating a web lexicon

In this “crash course” lesson to the web platform, we will walk through the development of a simple but effective bilingual lexicon.

Along the way, you will learn what the Web Platform is, and be introduced to its key technologies: HTML, CSS, JSON, and Javascript. At the end of the lesson we will see some options for how the document can be deployed to the internet and shared with colleagues and friends.

Building a searchable Boasian database

One drawback of the current suite of software tools in use in language documentation is the fact that different software tools use distinct data models. This necessitates regimes of input/output, and consequently, of context switching as linguists are working.

In this lesson we will learn what data modeling is, and how a simple model can cover most of the types of content that constitute the Boasian trilogy.
